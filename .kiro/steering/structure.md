# Project Structure

```
kiro-rider-light/
  package.json                  # VS Code extension manifest
  themes/
    Kiro Rider Light-color-theme.json  # Generated output -- do not edit by hand
  theme_gen/                    # Python theme generator (source of truth)
    main.py                     # Entry point: assembles and writes the theme JSON
    pyproject.toml              # Dependencies, tool config (ruff, mypy, pylint)
    uv.lock                     # Pinned lockfile (commit this)
    core/
      tcol.py                   # TCol: OKLCH color type with contrast/alpha helpers
      font_style.py             # FontStyle enum (bold, italic, underline)
      hue_series.py             # Hue harmony utilities
    palette/
      palette.py                # Palette dataclass: all seed + derived colors for a variant
      syntax.py                 # SyntaxPalette: the 17 named syntax roles
      theme.py                  # Theme: composes Palette + SyntaxPalette
      editor.py                 # EditorPalette helpers
    lang/
      protocol.py               # Language protocol + TokenColorRule + SemanticTokenStyle
      base.py                   # BaseSyntax: shared TextMate rules across all languages
      registry.py               # LanguageRegistry: merges all lang rules into final lists
      semantic.py               # GlobalSemanticTokens: cross-language semantic rules
      <lang>.py                 # Per-language TextMate + semantic overrides
    ui/
      protocol.py               # UISection protocol: build(theme) -> dict[str, TCol]
      composer.py               # ColorMapComposition: merges all UI sections
      <section>.py              # Per-section VS Code color key mappings
    tests/                      # pytest suite
      fixtures/                 # Snapshot JSON for output regression tests
  examples/                     # Sample source files for screenshot/visual testing
    src/main/<lang>/generic.*   # One file per supported language
  references/                   # Upstream docs (VS Code theme colors, scope naming, etc.)
  scripts_old/                  # Legacy extraction/analysis scripts -- not part of build
  img/                          # Screenshots and icons for marketplace listing
```

## Key Conventions

- `themes/Kiro Rider Light-color-theme.json` is always generated by `uv run main.py` -- never edit it directly.
- All color values flow through `TCol` (OKLCH). Never hardcode hex strings in generator code; derive from `Palette` or `SyntaxPalette`.
- Each language module implements `BaseLanguage` (TextMate rules) and optionally overrides `semantic_token_overrides`.
- Each UI section implements `UISection` protocol: `build(theme) -> dict[str, TCol]`.
- Tests include a snapshot test (`test_snapshot.py`) -- regenerate fixtures when intentional output changes occur.
- `scripts_old/` is archived reference material; do not add new scripts there.
